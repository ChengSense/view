插值

插值用于把绑定相应变量的内容以文本的格式输出到视图和其他的框架一样用双大括号来表示。

语法 

    
    {{ 表达式 }}

示例 

    
    <span id="msg">Message: {{ msg }}</span>
    
    
    
    new View({
        view: "span#msg",
        model: {
            msg: "输入点什么?"
        }
    });
    
    
    
    输入点什么?

插值HTML

插值指令默认的情况下会将绑定的内容解析成文本的格式；在某些情况下需要向视图输出HTML代码，默认的插值指令不会得到你想要的结果。 这里可以使用插值HTML指令。HTML代码里同样可以使用各种指令，View可以将HTML代码纳入其的生命管理周期。

语法

    
    {{> show }}

示例

    
    <span id="msg">Message: {{> show }}</span>
    
    
    
    new View({
        view: "span#msg",
        model: {
            show "<div>{{msg}}</div>"
            msg: "输入点什么?"
        }
    });
    
    
    
    输入点什么?

关于{{ }}和{{> }}为什么不合并成统一的一个，{{> }}指令如果滥用可能会有HTML注入的风险，分成两个指令这样可以轻松的控制输出的粒度。 可以提升控制的粒度，同时可以避免HTML注入的风险，另外解析文本比解析HTML的代价小很多，有助于性能的提升，细分成两个乐而不为呢。

条件 @when

View里的条件指令，相对于其他的js框架的条件的指令更容易理解（View的诞生其实和AngularJS糟糕难用的ng-if指令，多少有点关系。 在某个风和日丽的上午或者下午，当然现在记不太清了，当用到ng-if指令实现某个效果的时候，被坑了一些时间，现在想想当时心里估计有1w个曹尼玛奔腾而过， 当时就在想设计ng-if指令的人脑子是不是坏掉了，已于常人难以理解。于是对其好感度急剧下降，在想有没有一套更简单的前端框架，百度，谷歌然而并没有找到； 随后在某个晚上开始撸起代码）。

语法

    
    @when(布尔表达式,{
        业务逻辑
    })

示例

    
    <div id="msg">
        @when(show,{
            <span>Message: {{ msg }}</span>
        })
    </div>
    
    
    
    new View({
        view: "div#msg",
        model: {
            msg: "输入点什么?",
            show : true
        }
    });
    
    
        
        输入点什么?

条件分支 @else

@else 要配合@when 指令一起使用

语法

    
    @when(布尔表达式,{
        业务逻辑
        @else
        业务逻辑
    })

示例

    
    <div id="msg">
        @when(show,{
            <span>Message: {{ msg }}</span>
            @else
            hello wold ！
        })
    </div>
    
    
    
    new View({
        view: "div#msg",
        model: {
            msg: "输入点什么?",
            show : true
        }
    });
    
    
    
    hello wold ！

由于View没有提供，将来也可能不会提供 else if 类似的语法；变通的可以嵌套多个 @when( ,{ @else }) 同样可以达到 else if 一样的效果。View的理念就是简单就好，尽量减少指令的数量， 减少开发者的脑容量，把学习成本降到最低。

循环 @each

语法

    
        @each(变量：主键(可以省略)：数组||对象,{
            业务逻辑
        })
    
    
    
        <ul id="ul">
            @each(li:ul,{
                <li>
                    {{ li.text }}
                </li>
            })
        </ul>
    
    
    
    new View({
        view: "ul#ul",
        model: {
            ul: [{
                text :"View.js"
            },{
                text :"Vue.js"
            },{
                text :"React.js"
            }]
        }
    });
    
    
    
        
    
            
  • View.js
    • 
            
  • Vue.js
    • 
            
  • React.js
    •

注意点，循环表达式的参数 “变量：数组||对象”，如需要对象的主键这可以写成 “变量：主键(可以省略)：数组||对象” 主键的定义不是必须的，不需要的话可以省略

循环属性 each

循环属性 each请参考 @each指令用法并无不同

语法

    
        <div each="item:key:list"></ldiv>

示例

    
    <ul id="ul">
        <li each="text:index:ul">
            {{ index + 1 }}. {{ text }}
        </li>
    </ul>
    
    
    
    new View({
        view: "ul#ul",
        model: {
            ul: ["View.js","Vue.js","React.js"]
        }
    });
    
    
    
    
    
        
  • 1.View.js
    • 
        
  • 2.Vue.js
    • 
        
  • 3.React.js
    •

多层嵌套

多层嵌套，通过复杂的指令组合，完成想你所想

示例

    
    @each(item:ul,{
        <ul id="ul">
            <li each="text:index:ul">
                @when(index==1,{
                    {{index + "." + text }}
                    @else
                    {{index}}. hello wold ！
                })
            </li>
        </ul>
    })
    
    
    
    new View({
        view: "ul#ul",
        model: {
            ul: ["View.js","Vue.js"]
        }
    });
